# CLAUDE.md

이 프로젝트에서는 아래 팀 규칙을 따릅니다:

## 팀 작업 규칙
`.claude/rules/TEAM.md` 파일의 규칙을 준수해주세요.

### 주요 원칙
- **Plan → Code → Review** 순서로 작업
- 한국어로 응답 (코드는 영어)
- AI는 초안 제안만, 최종 결정은 사람이
- 사용자가 거부한 방법은 다시 제안하지 않음

### 중요 금지사항
- `git push --force` 금지
- 운영 브랜치 직접 작업 금지
- 새 라이브러리 추가보다 기존 코드 재사용
- 요청 범위 외 문제는 자동 수정하지 않고 알림만

## 🔐 Git 관련 규칙

### 1. 기본 원칙
- 모든 커밋은 명확한 목적을 가져야 한다.
- 하나의 커밋에는 하나의 변경 목적만 포함한다.
- 의미 없는 커밋 메시지(`update`, `fix bug` 등)는 금지한다.

---

### 2. Claude 사용 시 규칙
- Git commit 시 Claude 서명(Generated by Claude, AI Assistant 등)을 절대 포함하지 않는다.
- Claude는 커밋 메시지 제안만 수행하며, 실제 커밋은 사람이 최종 확인 후 수행한다.
- Claude가 커밋 메시지를 생성할 경우, 반드시 아래 커밋 컨벤션을 따른다.

---

### 3. Git Author 설정 (필수)
모든 커밋은 아래 Author 정보로 설정한다.

- Name: shinseunguk
- Email: krdut1@gmail.com

git config --global user.name "shinseunguk"
git config --global user.email "krdut1@gmail.com"

---

### 4. Commit Message Convention (필수)
모든 커밋 메시지는 아래 형식을 반드시 따른다.

<type>(<scope>): <subject>

feature(type): <subject>

---

### 5. Commit Message 예시
feat(auth): 소셜 로그인 기능 추가
feature(reservation): 부스 예약 기능 구현
fix(queue): 대기열 순서 오류 수정
refactor(api): 응답 DTO 구조 개선

---

### 6. 금지 예시
add login feature
feat: 로그인 추가함
update
fix bug
